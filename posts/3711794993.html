001.原理-事件循环

糖丸2023-06-152023-11-01

事件循环

浏览器的进程模型

何为进程？

程序运行需要有它自己专属的内存空间，可以把这块内存空间简单的理解为进程。

  • 每个应用至少有一个进程，

    1. 进程之间相互独立，🪶即内存空间之间是相互隔离的。

      1. 🎭如上图示

        • QQ、微信、王者他们的内存空间有的多、有的少，
        • 但是内存没有交叉，也不可能交叉。

      2. 设计目的

        • 为了隔离，假设微信出现问题不会影响到QQ、王者的运行。

    2. 当然内存之间是可以通信的，但需要双方同意。

何为线程？

有了进程后，就可以运行程序的代码了。

  • 运行代码的『人』称之为『线程』。

  • 一个进程至少有一个线程，

    • 所以在进程开启后会自动创建一个线程来运行代码，该线程称之为主线程。
    • 🔔提示
      • 如果主线程结束，那么整个程序就停止运行了。

  • 如果程序需要同时执行多块代码，那么主线程就会启动更多的线程来执行代码，

    • 所以一个进程中可以包含多个线程。

🔔提示

  • 以上线程共用同一个进程。

浏览器有哪些进程和线程？

浏览器是一个多进程多线程的应用程序，浏览器内部工作极其复杂。

  1. 为了避免相互影响，
  2. 为了减少连环崩溃的几率，
    • 当启动浏览器后，它会自动启动多个进程。

🔔提示

  • 可以在浏览器的任务管理器中查看当前的所有进程。

其中，最主要的进程有

  1. 浏览器进程

    • 主要负责界面显示、用户交互、子进程管理等。
      • 浏览器进程内部会启动多个线程处理不同的任务。

  2. 网络进程

    • 负责加载网络资源。
      • 网络进程内部会启动多个线程来处理不同的网络任务。

  3. 渲染进程

    • 渲染进程启动后，会开启一个渲染主线程，

      • 主线程负责执行HTML、CSS、JS 代码。

    • 默认情况下，浏览器会为每个标签页开启一个新的渲染进程，以保证不同的标签页之间不相互影响。

    • 🔔提示

      • 将来该默认模式可能会有所改变，
        • 如有兴趣可参见chrome官方说明文档。

渲染主线程是如何工作的？

渲染主线程是浏览器中最繁忙的线程，需要它处理的任务包括，但不限于 ➾

  1. 解析 HTML
  2. 解析 CSS
  3. 计算样式
  4. 布局
  5. 处理图层
  6. 每秒把页面画 60 次『FPS』
  7. 执行全局 JS 代码
  8. 执行事件处理函数
  9. 执行计时器的回调函数
  10. …

思考题: 为什么渲染进程不适用多个线程来处理这些事情？

🟩解答: 浏览器做不到。

要处理这么多的任务，主线程遇到了一个前所未有的难题: 如何调度任务？

  • 🍒例如
    1. 我正在执行一个 JS 函数，执行到一半的时候，用户点击了按钮，我该立即去执行点击事件的处理函数吗？
    2. 我正在执行一个 JS 函数，执行到一半的时候，某个计时器到达了时间，我该立即去执行它的回调吗？
    3. 浏览器进程通知我“用户点击了按钮”，与此同时，某个计时器也到达了时间，我该处理哪一个呢？
    4. …

渲染主线程想出了一个绝妙的主意来处理这个问题: 排队

  1. 在最开始的时候，渲染主线程会进入一个无限循环，

  2. 每一次循环会检查消息队列中是否有任务存在。

    1. 如果有，
      • 则取出第一个任务执行，执行完一个后进入下一次循环。
    2. 如果没有，
      • 则进入休眠状态。

  3. 其他所有线程（包括其他进程的线程）可以随时向消息队列添加任务，新任务会加到消息队列的末尾。

    • 在添加新任务时，如果主线程是休眠状态，则会将其唤醒以继续循环拿取任务。
    • 这样一来，就可以让每个任务持续的、有条不紊的进行下去了。

  4. 整个过程，被称之为事件循环（消息循环）。

若干解释

何为异步？

代码在执行过程中，会遇到一些无法立即处理的任务，

  • 🍒例如
    1. 计时完成后，需要执行的任务 ⥤ setTimeout、setInterval。
    2. 网络通信完成后，需要执行的任务 ⥤ XHR、Fetch。
    3. 用户操作后，需要执行的任务 ⥤ addEventListener。

如果让渲染主线程等待这些任务的时机达到，就会导致主线程长期处于『阻塞』的状态，从而导致浏览器『卡死』。

  1. 同步

同步举例

  • 有一家餐馆只有一个服务员，这时来了三个客人，
    • 服务员让一个客人点完菜以后，就等厨师将菜品制作完成，再去服务下一个客人，
    • 这就会造成另外两个客人不耐烦，从而拉低餐馆的营业额。

❗注意

  • 渲染主线程承担着极其重要的工作，无论如何都不能阻塞！
    • 因此，浏览器选择异步来解决这个问题。

  1. 异步

    异步举例

    • 有一家餐馆只有一个服务员，这时来了三个客人，
      • 服务员让一个客人点完菜以后，直接服务下一个客人，从而提高餐馆的营业额。

    🔔提示

    • 使用异步的方式，渲染主线程将永不阻塞。

面试题: 如何理解 JS 的异步？

参考答案:

JS是一门单线程的语言，

  • 这是因为JS运行在浏览器的渲染主线程中，而渲染主线程只有一个。

  • 而渲染主线程承担着诸多的工作，🍒例如: 渲染页面、执行JS… 都在其中运行。

如果使用同步的方式，就极有可能导致主线程产生阻塞，

  • 从而导致消息队列中的很多其他任务无法得到执行。
    1. 这样一来，一方面会导致繁忙的主线程白白消耗时间，
    2. 另一方面导致页面无法及时更新，给用户造成卡死现象。

所以浏览器采用异步的方式来避免。

  • 具体做法

    • 当某些任务发生时，比如计时器、网络、事件监听，

      • 主线程将任务交给其他线程去处理，自身立即结束任务的执行，转而执行后续代码。
      • 当其他线程完成时，将事先传递的回调函数包装成任务，
        • 加入到消息队列的末尾排队，等待主线程调度执行。

    • 在这种异步模式下，浏览器将永不阻塞，从而最大限度的保证了单线程的流畅运行。

任务有优先级吗？

🟩解答

  • 任务没有优先级，在消息队列中先进先出，但消息队列是有优先级的。

    • 根据 W3C 的最新解释 ➾

      1. 每个任务都有一个任务类型，

        1. 🔷分类

          1. 相同类型的任务必须在一个消息队列，
            • 🍒例如
              • 两个点击事件必须在同一个消息队列。
          2. 不同类型的任务可以分属于不同的消息队列，也可以分属于相同消息队列。
            • 🍒例如
              • 网络任务和计时器任务在同一个消息队列。

        2. 在一次事件循环中，浏览器可以根据实际情况，从不同的队列中取出任务执行。

      2. 浏览器必须准备好一个微队列，微队列中的任务优先于所有其他任务执行。

      引用站外地址
      whatwg.org
      官方解释

    • 🔔提示

      • 随着浏览器的复杂度急剧提升，W3C 不再使用宏队列的说法。

在目前 chrome 的实现中，至少包含了下面的队列

  1. 延时队列: 用于存放计时器到达后的回调任务，优先级『中』。
  2. 交互队列: 用于存放用户操作后产生的事件处理任务（🍒例如: 鼠标、键盘事件），优先级『高』。
  3. 微队列: 用户存放需要最快执行的任务，优先级『最高』。

  1. 面试题: 阐述一下 JS 的事件循环。

    参考答案

    事件循环又叫做消息循环，是浏览器渲染主线程的工作方式。

    1. 在Chrome浏览器的源码中，它开启了一个不会结束的 for 循环（死循环），
      • 每次循环从消息队列中，取出第一个任务执行，
      • 而其他线程只需要在合适的时候，将任务加入到队列末尾即可。
    2. 过去把消息队列简单分为宏队列和微队列，
      • 这种说法目前已无法满足复杂的浏览器环境，取而代之的是一种更加灵活多变的处理方式。
    3. 根据 W3C 官方的解释，每个任务有不同的类型，
      1. 相同类型的任务必须在同一个队列，
      2. 不同类型的任务可以分属于不同的消息队列，也可以分属于相同消息队列。
        • 不同任务队列有不同的优先级，
          • 在一次事件循环中，由浏览器自行决定取哪一个队列的任务。
          • 但浏览器必须有一个微队列，
            • 微队列的任务一定具有最高的优先级，必须优先调度执行。

  2. 面试题: JS 中的计时器能做到精确计时吗？为什么？

    参考答案

    不行，因为计算机硬件没有原子钟（一种计时装置），无法做到精确计时。

    1. 操作系统的计时函数本身就有少量偏差，

      • 由于 JS 的计时器最终调用的是操作系统的函数，也就携带了这些偏差。

    2. 按照 W3C 的标准，浏览器实现计时器时，

      • 如果嵌套层级超过 5 层，则会带有 4 毫秒的最少时间，

      • 这样在计时时间少于 4 毫秒时，又带来了偏差。

        • 🔥案例

    3. 受事件循环的影响，计时器的回调函数只能在主线程空闲时运行，因此又带来了偏差。
