为什么 transform 的效率高？

📌原因

  • transform（元素转换） 既不会影响布局，也不会影响绘制指令，

    • 它影响的只是渲染流程的最后一个『draw』阶段。

    • 由于 draw 阶段在合成线程中，所以 transform 的变化几乎不会影响渲染主线程。

      • 反之，渲染主线程无论如何忙碌，也不会影响 transform 的变化。

    • 而使用left却会造成，造成浏览器回流重排。

      1. 设置left ➾ 合成图层

        • 🔥案例

          1. 添加left: 0;

          2. 删除left: 0;

      2. 设置translate ➾ 独立图层

        • 🔥案例

Css3 transform 属性

用于元素的2D或3D转换。

  • 🪶即使得元素可以进行移动、缩放、旋转、倾斜等操作。

    • ❗注意
      • transform属性只能添加给块元素或行内块元素，不能添加给行内元素。

  • 🎯特性

    默认值none
    继承no
    JavaScript 语法object.style.transform=“rotate(7deg)”

  • 语法

    • 🍬解析
      1. none ⥤ 不进行元素转换。
      2. transform-functions ⥤ 一个或多个元素的2D或3D转换函数。

transform 多个转换函数的使用方法

  1. transform属性添加单个转换函数时 ➾ 

    • transform: rotate(45deg);

  2. transform属性添加多个转换函数时，需用{空格隔开} ➾ 

    • transform: translate(50px, 90px) scale(2) rotate(45deg);

    • ❗注意

      1. transform属性添加多个转换函数时，

        • 执行顺序 ➾ 从后往前执行。

      2. translate() 会受到scale()、rotate()、skew()的影响。

        1. scale() 对translate() 的影响。

          1. 元素先缩小一半，在向右平移50px。『推荐』

          2. 元素先向右平移50px，在缩小一半。

          3. 综述『推荐 ➾ 先书写translte()，再书写scale()』

        2. rotate()对translate()的影响。

          1. 元素先顺时针旋转90度，在向右平移50px。『推荐』

          2. 元素先向右平移50px，在顺时针旋转90度。

          3. 综述『推荐 ➾ 先书写translte()，再书写rotate()』

2D Transform函数

translate()

将元素按照指定的方向进行2D水平和垂直方向平移，

  • 类似于position: relative； + left|right、top|bottom: ?;。

  • 语法

    • 🍬解析

      1. x ⥤ 元素沿着水平方向（X轴）移动的距离，
      2. y ⥤ 元素沿着垂直方向（Y轴）移动的距离。

    • 🌿口诀

      1. 水平方向
        • 正值表示向右平移，负值表示向左平移。『正右负左』
      2. 垂直方向
        • 正值表示向下平移，负值表示向上平移。『正下负上』

    • ⏳详述

      1. 沿水平方向移动
        1. 向左移动{translate(-x, 0)}
        2. 向右移动{translate(x, 0)}
      2. 沿垂直方向移动
        1. 向上移动{translate(0, -y)}
        2. 向下移动{translate(0, y)}
      3. 沿对角方向移动『逆时针』
        1. 向左上角移动{translate(-x, -y)}
        2. 向右上角移动{translate(x, -y)}
        3. 向右下角移动{translate(x, y)}
        4. 向左下角移动{translate(-x, y)}

    • 🔥案例

      • ❗注意

        • y是一个『可选参数』，

          • 若translate函数只设置了一个参数，
            • 省略了另一个参数，则第二个参数默认值 ➾ 0。
              • 🪶即元素只在水平方向上平移，垂直方向上不平移。

        • 🔥案例

          • 等价于

        • 🔥案例{转换函数应用%}

          • 向左移动100px。
    • transform属性的转换函数应用%，将相对于应用元素的宽高。

rotate()

将元素按照指定的角度进行2D旋转。

  • 语法

    • 🍬解析

      • ndeg ⥤ 元素旋转的角度{(0 ⥤ 360)deg}，
      • 🔔提示
        • deg ⥤ 角度单位。

    • 🌿口诀

      • 正值表示顺时针旋转，负值表示逆时针旋转。 『正顺负逆』

    • 🔥案例

Css3 transform-origin 属性

默认情况下，Css转换{移动、缩放、旋转、倾斜等操作}都是以元素基点（中心点）位置进行变形的。

  • 而Css3中的transform-origin属性用于设置转换元素的基点位置。

    • 🔔提示
      1. 使用此属性必须使用transform 属性。
        • ❗注意
          • 在同一个类中，transform-origin属性的书写位置应该先于transform属性。
      2. 2D转换元素可以改变元素的{X轴和Y轴}。
      3. 3D转换元素可以改变元素的{X轴、Y轴和Z轴}。

  • 🎯特性

    默认值50% 50% 0
    继承no
    JavaScript 语法object.style.transformOrigin=“20% 40%”

  • 语法

    • 🍬解析

      1. x-axis『水平方向』
        • 定义转换元素被置于 X 轴的何处。
        • 可能的值 ➾ 
          1. left{0%}
          2. center{50%}
          3. right{100%}
          4. length
          5. %
      2. y-axis『垂直方向』
        • 定义转换元素被置于 Y 轴的何处。
        • 可能的值 ➾ 
          1. top{0%}
          2. center{50%}
          3. bottom{100%}
          4. length
          5. %
      3. z-axis『Z轴偏移量』
        • 定义转换元素被置于 Z 轴的何处。
        • 可能的值 ➾ 
          • length{默认值 ➾ 0}
      • ⏳详例
        1. left top丨top left ⇌\rightleftharpoons⇌ 0 0丨0% 0%
        2. top丨top center丨center top ⇌\rightleftharpoons⇌ 50% 0
          • 🔔提示
            • 如果只取一个值，表示垂直方向值为默认值 ➾ center

    • 🔥案例

      1. 默认值，以转换元素自身基点旋转45deg。

      2. 以顶部旋转180deg。

      3. 🟥错误案例

        • transform-origin不能写在这里{❗}。
        • 📌原因
          • 此处效果为 ➾ 
            • 先按照中心旋转，当触发伪类时，才按照顶部中心点旋转。
            • transform-origin属性的书写位置应为当前元素，而不是当前元素的伪类。

      4. 折扇效果案例

scale()

将元素按照指定的倍数进行2D水平和垂直方向缩放 ➾ X轴和Y轴同时缩放，🪶即改变元素的宽度和高度。

  • 语法

    • 🍬解析

      1. x ⥤ 元素沿着水平方向（X轴）缩放的倍数，

      2. y ⥤ 元素沿着垂直方向（Y轴）缩放的倍数。

        • ❗注意

          1. y是一个『可选参数』，

            • 若scale函数只设置了一个参数，

              • 省略了另一个参数，则第二个参数将等于第一个参数。

            • 🔥案例
