发表于2023-05-30更新于2023-07-25

多列布局

瀑布流，又称瀑布流式布局。

  • 📌原因 !

    • 因为视觉上就是像瀑布一样的波澜不绝，参差不齐。

  • 🔉说明

    • 瀑布流是一种多列等宽不等高的页面展示方式。
      • 当我们把按下滚动键上下滚动的时候，瀑布流式布局就会慢慢的往下加载数据到尾部。
优点
  1. 可以节省空间。很有艺术性质。对于触屏设备非常友好，既可以往下也可以往下浏览。
    • 用户浏览时有观赏性思维，不容易被打断，更容易获得点击。
  2. 从客户角度的心理看女性是一种喜欢买买买逛街的，而看不到尽头。
    • 瀑布流非常适合这种心理。
  3. 瀑布流的图片就像商品，像买东西，像购物一样，
    • 女性在很多陈列柜里面寻找自己想要的东西，像一条没有尽头的河流。
缺点
  1. 确认一下不知道内容的长度，对内容没有宏观掌控，想找回刚才浏览地方，找不到具体位置。
  2. 容易造成页面加载负荷。
  3. 可能造成用户浏览疲劳，没有休息时间。
📿适用场景

  1. 内容都是以视频图片为主，视频也是以图片展示为主。

    • 当我们的鼠标点击到快移到图片那里，图片就会变成视频模式，图片的占用空间很大。
    • 而且大脑处理速度比文字要快，同时现在可以看到很多内容，也可以理解到很多内容。
      • 如果用分页展示的话，用户会频繁翻页，影响体验，而瀑布流可以解决这个问题。

  2. 信息与信息之间是独立的，不过你有更好的选择。

    • 如果信息关联强，用户会进行回访操作，查看之前或者之后的信息，

      • 相反，假如信息是独立的话。如果可以接受不一样的信息。

    • 当信息和搜索内容比较模糊的时候，瀑布流是更好的选择。

      • 瀑布流给人的印象是一种直观的印象，
        • 同时展示信息，和用户搜索的要求是大致一样的。
      • 而分页展示的形象是越靠上的信息越匹配。
        • 与用户搜索的匹配信息与信息之间没有明显分度就可以用瀑布流。

  3. 用户目的性不强的时候，瀑布流是更好的选择。

    • 如果用户有特定需要查找的信息，分页查找定位更方便，
      • 而当目的性较弱的时候，瀑布流可以增加用户停留的时间和意想不到的收获。

    1. 列宽与列数的设置，很容易产生冲突！

      2. 如果同时设置了这两个属性，则他们会产生冲突。

          • 🪶即谁设置的宽度值大，则谁生效。
    2. 为了解决兼容问题，最好加上浏览器前缀。

column-count

column-count 属性用来设置将元素分为几列。

  • 🎯特性
默认值auto
继承no
JavaScript 语法object.style.columnCount=3
  • 属性值
值描述
number使用具体数值将元素划分为指定的列数。
auto由其他属性决定具体的列数，🍒例如: column-width。
  • 🔥案例
    • column-count: 4;

column-fill

column-fill 属性用来设置如何填充每个列。

  • 🪶即设置每个列的高度是否统一。

  • 🎯特性

默认值balance
继承no
JavaScript 语法object.style.columnFill=“auto”
  • 属性值
值描述
balance平衡模式，根据其他列属性值使列的高度尽可能平衡。
auto自动分配模式，按顺序对每个列进行填充，列的高度会各有不同。

  • 🔔提示

    1. 平衡模式也叫内容均匀分配模式，

      • 该模式中浏览器会调整各个列的高度， 使各个列的内容的高度基本一致。

    2. 自动分配模式会使内容按顺序分配在各个列中，

      • 所以各个列的高度可能不一样。

  • 🔥案例

column-gap

column-gap 属性用来设置列与列之间的间隙。

  • 🎯特性
默认值normal
继承no
JavaScript 语法object.style.columnGap=“40px”
  • 属性值
值描述
length将列与列之间的间隔设置为指定的宽度。
normal将列与列之间的间隔设置为与 font-size 属性相同的大小，🪶即 1em。
  • 🔥案例

column-rule

column-rule 属性是一个简写属性，

  • 它与 border 属性非常相似，用来设置列与列之间边框的宽度、样式和颜色。

  • 🎯特性

默认值medium none black
继承no
JavaScript 语法object.style.columnRule=“3px outset #ff00ff”
  • 语法
  • 属性值
值说明
column-rule-width设置列与列之间边框的宽度规则。
column-rule-style设置列与列之间边框的样式规则。
column-rule-color设置列与列之间边框的颜色规则。
  • column-rule-style的可选值。
值描述
none不定义边框样式。
hidden隐藏边框样式。
dotted定义点状边框。
dashed定义虚线边框。
solid定义实线边框。
double定义双实线边框。
groove定义 3D grooved 边框，边框效果取决于宽度和颜色值。
ridge定义 3D ridged 边框，边框效果取决于宽度和颜色值。
inset定义 3D inset 边框，边框效果取决于宽度和颜色值。
outset定义 3D outset 边框，边框效果取决于宽度和颜色值。

column-span

column-span 属性用来设置元素应该跨越多少列。

  • 🎯特性
默认值none 或 1
继承no
JavaScript 语法object.style.columnSpan=“all”
  • 属性值
值描述
none 或 1表示元素不跨越列。
🔔提示: 不能是其他数值。
all表示元素横跨所有列。

  • 🔔提示

    • 我们常将标题横跨所有列，即column-span: all;。

  • 🔥案例

column-width

column-width 属性用来设置每个列的宽度。

  • 🎯特性
默认值auto
继承no
JavaScript 语法object.style.columnWidth=“100px”
  • 属性值
值描述
auto由浏览器决定列的宽度。
length为每个列指定具体的宽度。
  • 🔥案例

columns

columns 属性是一个简写属性，用来同时设置列的宽度和列的数量。

  • 🎯特性
默认值auto auto
继承no
JavaScript 语法object.style.columns=“100px 3”
  • 语法
  • 🔔提示

    1. 其中 column-width 表示每个列的宽度。

    2. column-count 表示列的数量。

    3. 🧮计算公式

      • 列宽 ＝（父元素宽度 ➖（列数-1) × 列间隙）➗ 列数

      • 🍒例如

        • 父元素40em，分为4列，列间隙（column-gap）为2em。
        • 那么列宽为(40−(4−1)×2)4=8.5em\frac{\left ( 40- \left ( 4-1 \right ) \times 2 \right ) }{4} = 8.5em4(40−(4−1)×2)​=8.5em。

      • 🔔提示

        1. 如果显式设置的列宽小于这个宽度，它会以这个宽度展示。
        2. 如果显式设置的列宽大于这个宽度，它会按照合适的尺寸重新排列列宽和列数。
          • 🪶即当前设置的列宽和列数并不绝对。
