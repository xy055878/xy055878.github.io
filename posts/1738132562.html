Css3 3D 转换

什么是3D的场景呢？

2D场景

  • 2D场景的坐标轴是由X、Y两条轴组成的{平面空间}。

3D场景

  • 3D场景的坐标轴是由X、Y、Z三条轴组成的{立体空间}，

    • 🔔提示

      • X轴、Y轴和Z轴正向分别朝向右、下和屏幕外。

    • 🎭图示

2D转换与3D转换间的不同之处

Css3 perspective 属性

用于定义3D元素距视图的距离，以像素{px}计。『用于父元素』

  • 🔔提示

    1. perspective属性只影响 3D 转换元素。
    2. 当元素添加perspective属性时，其子元素会获得透视效果，而不是元素本身。
    3. 请与perspective-origin属性同使用该属性，这样就能够改变3D元素的位置。

  • ❗注意

    • 透视{perspective}是3D转换中，最重要的内容。

      • 若不设置透视，元素的3D变形效果将无法实现。

        • 🎭图示

        • 由如上三个图可得出，

          • 如果不设置透视，那么浏览器会将元素的3D转换操作投射到2D视平面上，
            • 最终呈现出来的只是元素的{宽度和高度}变化。

      • 要深入了解透视，需要了解{观察者、被透视元素和变形{转换}元素}这几个概念。

        1. 观察者
          • 浏览器模拟出来用来观察{被透视元素}的一个没有尺寸的点，
          • 观察者发出的{视线}，就类似于{一个点}向光源发出{光线}。
        2. 被透视元素
          • 被观察者观察的元素，根据属性设置的不同，它有可能是 ➾ 
            • 变形{转换}元素本身或变形{转换}元素的父级元素或祖先元素。
          • 主要进行{perspective、perspective-origin等属性}的设置。
        3. 变形{转换}元素
          • 顾名思义，就是进行transform3D转换的元素，
          • 主要进行{transform、transform-origin、backface-visibility等属性}的设置。

      • 🎭图示

      • 透视距离{perspective}是指观察者沿着{平行于}Z轴的视线与屏幕之间的距离，

        • 🪶即元素距离视图的距离，简称视距。

  • 🎯特性

    默认值none
    继承no
    JavaScript 语法object.style.perspective=500

  • 语法

    • 🍬解析

      1. none
        • 不设置透视，与0等价。
      2. number
        • 元素距离视图的距离，以像素计{px}。
        • ❗注意
          1. perspective属性值，不可取{0或负数}，
            • 📌原因
              • 观察者与屏幕距离为 ➾ 0时或在屏幕背面{负数}时，
              • 是不可以观察到{被透视元素}的正面的。
          2. perspective属性值，不可取%。
            • 📌原因
              • 百分比需要相对的元素，但Z轴并没有可相对的元素尺寸。
        • 一般地，物体离得越近，显得越大，物体离得越远，显得越小。『近大远小』

          • 反映在perspective属性上，

            1. 就是该属性值越大，元素的3D效果越不明显。
            2. 就是该属性值越小，元素的3D效果越明显。

          • 🍒例如

            1. 如果离一个人很近，甚至可以看到他的毛孔，
            2. 如果离一个人很远，可能只看到一个轮廓。

          • 🎭图示

    • 👻特别的

      • perspective: 1200px;和transform: perspective(1200px);的区别

      • 🎈用法

        1. perspective: 1200px;『在父元素中使用』
        2. transform: perspective(1200px);『在子元素中使用』

      • 🔔提示

        • 如上两个属性同时设置，将会发生冲突，
        • 建议只设置父元素，通常的数值在{900~1200}之间。

      • 🍬解析

        1. 透视属性perspective和透视函数perspective()
        • 都用于通过使Z轴上，
          1. {靠近}观察者的元素显得更大，
          2. {远离}观察者的元素显得更小。
        • 来赋予元素深度 ➾ 
          1. 视距的值越小，元素离观察者越近，效果越令人印象深刻，
          2. 视距的值越高，元素离屏幕越远，效果越微妙。
        1. 当您使用透视函数perspective()将透视应用于元素时，
        • 您仅在{该元素}上激活了三维空间，
          • 在对单个转换元素应用三维变换时，perspective()表示法很方便。
        • 但是当你在一个容器中，有多个转换元素时，
          • 如果每个元素都使用透视函数perspective()设置一个透视图，
            • 那么元素将不会按预期排列。
              • 📌原因
                • 它们每个都有自己的三维空间，因此也有自己的消失点。
            • 为避免这种情况并允许元素对齐，它们应该共享相同的空间。
              • 通过使用父容器上的透视属性perspective，
                • 创建一个由其所有子元素共享的三维空间，
                • 这些子元素都将使用3维变换进行变换。

        1. 🎭图示

      • 🍒例如

        1. 在本身元素上设置透视无效果。

        2. 在父级元素上设置透视有效果。

        3. 🎭图示

Css3 perspective-origin 属性

用于定义3D元素所基于的X轴和Y轴。『用于父元素』

  • 透视原点{perspective-origin}是指观察者的位置，

    • 一般地，

      1. 观察者位于与屏幕平行的另一个平面上，且观察者始终是与屏幕垂直的。

      2. 观察者的活动区域是被观察元素的盒模型区域。

    • 🎭图示

      ​

  • 🎯特性

    默认值50% 50%
    继承no
    JavaScript 语法object.style.perspectiveOrigin=“10% 10%”

  • 语法

    • 🍬解析
      1. x-axis
        • 定义该视图在X轴上的位置，默认值 ➾ 50%。
        • 可选值
          1. left
          2. center
          3. right
          4. length
            • 🔔提示
              • 表示在X轴上，离0点(元素边框外侧左上角)的偏移量。
          5. %
            • 🔔提示
              • 表示相对于{被透视元素}的宽度和，
              • 🪶即
                • width + 横向padding + 横向border。
      2. y-axis
        • 定义该视图在Y轴上的位置，默认值 ➾ 50%。
        • 可选值
          1. left
          2. center
          3. right
          4. length
            • 🔔提示
              • 表示在Y轴上，离0点(元素边框外侧左上角)的偏移量。
          5. %
            • 🔔提示
              • 表示相对于{被透视元素}的高度和，
              • 🪶即
                • height + 纵向padding + 纵向border。
    • 🔔提示
      1. 当只有一个值时，默认第二个值为center(50%)。
      2. perspective-origin属性必须定义在设置有perspective属性的元素上，
        • 🪶即必须定义在当前元素的{父元素或祖先元素}上。
    • ⏳详例
      1. left top丨top left ⇌\rightleftharpoons⇌ 0 0丨0% 0%
      2. top丨top center丨center top ⇌\rightleftharpoons⇌ 50% 0
        • 🔔提示
          • 如果只取一个值，表示垂直方向值为默认值 ➾ center。

  • 🔥案例

Css3 transform-origin 属性

在没有设置过transform-origin属性时， 『用于子元素』

  • Css转换进行的{移位、缩放、旋转}等操作都是以元素基点{中心点}位置进行转换的。

    • 而Css3中的transform-origin属性用于定义元素的基点位置。

  • 🎯特性

    默认值50% 50% 0
    继承no
    JavaScript 语法object.style.transformOrigin=“20% 40%”

  • 语法

    • 🍬解析

      1. x-axis
        • 定义该视图在X轴上的位置，默认值 ➾ 50%。
        • 可选值
          1. left
          2. center
          3. right
          4. length
            • 🔔提示
              • 表示在X轴上，离0点(元素边框外侧左上角)的偏移量。
          5. %
            • 🔔提示
              • 表示相对于{被透视元素}的宽度和，
              • 🪶即
                • width + 横向padding + 横向border。
      2. y-axis
        • 定义该视图在Y轴上的位置，默认值 ➾ 50%。
        • 可选值
          1. left
          2. center
          3. right
          4. length
            • 🔔提示
              • 表示在Y轴上，离0点(元素边框外侧左上角)的偏移量。
          5. %
            • 🔔提示
              • 表示相对于{被透视元素}的高度和，
              • 🪶即
                • height + 纵向padding + 纵向border。
      3. z-axis
        • 定义该视图在Z轴上的位置。
        • 可选值
          • length
            • 🔔提示
              • 表示在Z轴上，离0点(元素边框外侧左上角)的偏移量。

    • 🔔提示

      • 当只有一个值时，默认第二个值为center(50%)。

      • ⏳详例

        1. left top丨top left ⇌\rightleftharpoons⇌ 0 0丨0% 0%
        2. top丨top center丨center top ⇌\rightleftharpoons⇌ 50% 0
          • 🔔提示
            • 如果只取一个值，表示垂直方向值为默认值 ➾ center。

Css3 backface-visibility 属性

用于定义当元素以{背面}面向屏幕时，是否可见。『用于子元素』

  • 若在对元素进行转换{譬如: \text{: }: 旋转}且不希望看到该元素的背面时，

    • 则backface-visibility属性很有用。

  • 🎯特性

    默认值visible
    继承性no
    JavaScript 语法object.style.backfaceVisibility=“hidden”

  • 语法

    • 🍬解析
      1. visible
        • 元素背面是可见的。
      2. hidden
        • 元素背面是不可见的。

Css3 transform-style 属性

用于定义转换元素及其子元素，在3d空间中是否呈现。『用于父元素』

  • ❗注意

    • 当设置overflow: 非visible;或clip: 非auto;时，transform-style: preserve-3d;将失效。

  • 🎯特性

    默认值flat
    继承no
    JavaScript 语法object.style.transformStyle=“preserve-3d”

  • 语法

    • 🍬解析

      1. flat

        • 表示所有子元素在{2D平面}上呈现。

        • 🪶即

          • 如果对一个元素设置transform-style: flat;
            • 则该元素的所有子元素都将被平展到该元素的{2D平面}上进行呈现。
              • 从而使得沿着{X轴或Y轴}方向旋转该元素，
              • 将导致位于{正或负}Z轴上位置的子元素，
              • 显示在该元素的平面上，而不是它的{前面或后面}。

        • 🎭图示

      2. preserve-3d

        • 表示所有子元素在{3D空间}中呈现。

        • 🪶即

          • 若对一个元素设置transform-style: preserve-3d;
            • 则该元素的所有子元素，都将位于{3D空间}中。

3D Transform函数

translate3d()

将元素按照指定的方向进行3D{水平、垂直和屏幕方向}的平移，

  • 语法

    • 🍬解析

      1. x ⥤ 元素沿着水平方向{X轴}移动的距离，

        • 取值

          • 长度值丨百分比。
          • 🔔提示
            • 百分比相对于其{本身元素}的宽度。

        • 🌿口诀

          • 水平方向
            • 正值表示元素{向右}平移，负值表示元素{向左}平移。『正右负左』
      2. y ⥤ 元素沿着垂直方向{Y轴}移动的距离，

        • 取值

          • 长度值丨百分比。
          • 🔔提示
            • 百分比相对于其{本身元素}的高度。

        • 🌿口诀

          • 垂直方向
            • 正值表示元素{向下}平移，负值表示元素{向上}平移。『正下负上』
      3. z ⥤ 元素沿着屏幕方向{Z轴}移动的距离。

        • 取值

          • 长度值。

        • 🌿口诀

          • 屏幕方向
            • 正值表示元素{向外}平移，负值表示元素{向内}平移。『正外负内』
translateX()

将元素按照指定的方向进行3D{水平方向}的平移，

  • 语法

    • 🍬解析

      • n ⥤ 元素沿着水平方向{X轴}移动的距离。

      • 🔔提示

        1. 正值表示元素{向右}平移，负值表示元素{向左}平移。『正右负左』
        2. translateX(n)相当于translate3d(n, 0, 0)。
translateY()

将元素按照指定的方向进行3D{垂直方向}平移，

  • 语法

    • 🍬解析

      • n ⥤ 元素沿着水平方向{Y轴}移动的距离。

      • 🔔提示

        1. 正值表示元素{向下}平移，负值表示元素{向上}平移。『正下负上』
        2. translateY(n)相当于translate3d(0, n, 0)。
translateZ()

将元素按照指定的方向进行3D{屏幕方向}平移，

  • 语法

    • 🍬解析

      • n ⥤ 元素沿着水平方向{Z轴}移动的距离。

      • 🔔提示

        1. 正值表示元素{向外}平移，负值表示元素{向内}平移。『正外负内』
        2. translateZ(n)相当于translate3d(0, 0, n)。

rotate3d()

将元素按照指定的角度，进行3D{水平、垂直和屏幕方向}的旋转，

  • 语法

    • 🍬解析
      1. x ⥤ 元素围绕水平方向{X轴}旋转的值，
        • 取值
          • 0和1。
        • 🌿口诀
          • 正值表示向后旋转，负值表示向前旋转。 『正后负前』
      2. y ⥤ 元素围绕垂直方向{Y轴}旋转的值，
        • 取值
          • 0和1。
        • 🌿口诀
          • 正值表示向右旋转，负值表示向左旋转。 『正右负左』
      3. z ⥤ 元素围绕屏幕方向{Z轴}旋转的值，
        • 取值
          • 0和1。
        • 🌿口诀
          • 正值表示平面向右旋转，负值表示平面向左旋转。 『平面正右负左』
      4. ndeg ⥤ 元素旋转的角度{(0~360)deg}。
        • 🔔提示
          • deg ⥤ 角度单位。
rotateX()

将元素按照指定的角度，进行3D{水平方向}的旋转，

  • 语法

    • 🍬解析

      • ndeg ⥤ 元素围绕水平方向{X轴}旋转的角度。

      • 🌿口诀

        • 正值表示向后旋转，负值表示向前旋转。 『正后负前』

      • 🔔提示

        • rotateX(ndeg)相当于rotate3d(1, 0, 0, ndeg)。
rotateY()

将元素按照指定的角度，进行3D{垂直方向}的旋转，

  • 语法

    • 🍬解析

      • ndeg ⥤ 元素围绕垂直方向{Y轴}旋转的角度。

      • 🌿口诀

        • 正值表示向右旋转，负值表示向左旋转。 『正右负左』

      • 🔔提示

        • rotateY(ndeg)相当于rotate3d(0, 1, 0, ndeg)。
rotateZ()

将元素按照指定的角度，围绕屏幕方向{Z轴}旋转。

  • 语法

    • 🍬解析

      • ndeg ⥤ 元素围绕屏幕方向{Z轴}旋转的角度。

      • 🌿口诀

        • 正值表示平面向右旋转，负值表示平面向左旋转。 『平面正右负左』

      • 🔔提示

        • rotateZ(ndeg)相当于rotate3d(0, 0, 1, ndeg)。

  • 🎭图示

scale3d()

将元素按照指定的倍数，进行3D{水平、垂直和屏幕方向}的缩放。

  • 语法

    • 🍬解析

      1. x ⥤ 元素沿着水平方向{X轴}缩放的倍数，

      2. y ⥤ 元素沿着垂直方向{Y轴}缩放的倍数，

      3. z ⥤ 元素沿着屏幕方向{Z轴}缩放的倍数。

    • 🔔提示

      1. 默认值 ➾ 1 ⥤ 不缩放，0 ⥤ 元素消失。
      2. 如果是负值，则会反转缩放。
      3. 如果值小于1，就代表缩小{缩}，如果值大于1，就代表放大{放}。
