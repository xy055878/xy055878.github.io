021.3D转换

糖丸2023-09-262023-09-27

CSS3 3D 转换

什么是3D的场景呢？

2D场景

  • 2D场景的坐标轴是由X、Y两条轴组成的{平面空间}。

3D场景

  • 3D场景的坐标轴是由X、Y、Z三条轴组成的{立体空间}，

    • 🔔提示

      • X轴正向、Y轴正向、Z轴正向分别朝向右、下和屏幕外。

    • 🎭图示

2D转换与3D转换间的不同之处

Css3 perspective 属性

用于定义3D元素距视图的距离，以像素{px}计。

  • 🔔提示

    1. perspective属性只影响 3D 转换元素。
    2. 当元素添加perspective属性时，其子元素会获得透视效果，而不是元素本身。
    3. 请与perspective-origin属性同使用该属性，这样就能够改变3D元素的位置。

  • ❗注意

    • 透视{perspective}是3D转换中，最重要的内容。

      • 若不设置透视，元素的3D变形效果将无法实现。

        • 🎭图示

        • 由如上三个图可得出，

          • 如果不设置透视，那么浏览器会将元素的3D转换操作投射到2D视平面上，
            • 最终呈现出来的只是元素的宽高变化。

      • 要深入了解透视，需要了解{观察者、被透视元素和变形元素}这几个概念。

        1. 观察者
          • 浏览器模拟出来用来观察{被透视元素}的一个没有尺寸的点，
          • 观察者发出视线，类似于一个点向光源发出光线。
        2. 被透视元素
          • 被观察者观察的元素，根据属性设置的不同，它有可能是 ➾ 
            1. 变形元素本身，
            2. 它的父级或祖先元素。
          • 主要进行{perspective、perspective-origin等属性}的设置。
        3. 变形元素
          • 顾名思义，就是进行transform3D转换的元素。
          • 主要进行{transform、transform-origin、backface-visibility等属性}的设置。

      • 🎭图示

      • 透视距离{perspective}是指观察者沿着平行于Z轴的{视线与屏幕}之间的距离，简称视距。

  • 🎯特性

    默认值none
    继承no
    JavaScript 语法object.style.perspective=500

  • 语法

    • 🍬解析
      1. none
        • 不设置透视，与0等价。
      2. number
        • 元素距离视图的距离，以像素计{px}。
        • ❗注意
          1. perspective属性值，不可取{0或负数}，
            • 📌原因
              • 观察者与屏幕距离为 ➾ 0时或在屏幕背面时，
              • 是不可以观察到{被透视元素}的正面的。
          2. perspective属性值，不可取%。
            • 📌原因
              • 百分比需要相对的元素，但Z轴并没有可相对的元素尺寸。
        • 一般地，物体离得越远，显得越小。『近大远小』
          • 反映在perspective属性上，就是该属性值越大，元素的3D效果越不明显。
          • 🍒例如
            1. 如果离一个人很近，甚至可以看到他的毛孔，
            2. 如果离一个人很远，可能只看到一个轮廓。
    • 👻特别的

      • 使用

        1. perspective:1200px;『在父元素中使用』
        2. transform: perspective(1200px);『在子元素中使用』

      • 👁️‍🗨️分析

        • 如上两个属性同时设置，将会发生冲突，建议只设置父元素，
        • 通常的数值在{900~1200}之间，
          • 当你的视线距离物体足够远的时候，基本上就不会有近大远小的感觉。
          • 🪶即使用transform：tranlateZ();
            • 几乎不会有效果，可以理解为此时目标距离观察者无限远，
            • transform：tranlateZ();
              • 移动{100px、200px}视觉上，感受是几乎差不多的。

      • 🍒例如

        1. 在本身元素上设置透视无效果。

        2. 在父级元素上设置透视有效果。

        3. 🎭图示

Css3 perspective-origin 属性🏅①

用于定义3D元素所基于的X轴和Y轴。

  • 透视原点{perspective-origin}是指观察者的位置，

    • 一般地，观察者位于与屏幕平行的另一个平面上，观察者始终是与屏幕垂直的。

    • 观察者的活动区域是被观察元素的盒模型区域。

      • 🎭图示

        ​

  • 🎯特性

    默认值50% 50%
    继承no
    JavaScript 语法object.style.perspectiveOrigin=“10% 10%”

  • 语法

    • 🍬解析
      • x-axis
        • 定义该视图在X轴上的位置，默认值 ➾ 50%。
        • 可选值
          1. left
          2. center
          3. right
          4. length
            • 🔔提示
              • 表示在X轴上，离0点(元素边框外侧左上角)的偏移量。
          5. %
            • 🔔提示
              • 表示相对于{被透视元素}的宽度和(width + 横向padding + 横向border)。
      • y-axis
        • 定义该视图在Y轴上的位置，默认值 ➾ 50%。
        • 可选值
          1. left
          2. center
          3. right
          4. length
            • 🔔提示
              • 表示在Y轴上，离0点(元素边框外侧左上角)的偏移量。
          5. %
            • 🔔提示
              • 表示相对于{被透视元素}的高度和(height + 纵向padding + 纵向border)。
    • 🔔提示
      1. 当只有一个值时，默认第二个值为center(50%)。
      2. perspective-origin属性必须定义在设置有perspective属性的元素上，
        • 🪶即必须定义在当前元素的{父元素或祖先元素}上。
