015.vw和vh

发表于2023-06-06更新于2023-07-23
字数总计:2146阅读时长:7分钟 杭州

vm、vh、vmin、vmax

  • vm、vh、vmin、vmax是一种视口单位，也是相对单位。

    • 它相对于视口（Viewport）大小。
    • 🔔提示
      • 视口(Viewport)即为你的浏览器，实际显示内容的区域，
        • 换句话说，就是你的不包括工具栏和按钮的网页浏览器。

  • ⏳详述

    1. vw(viewport width) ⥤ 视口宽度的百分比，
      1. 1vw视口宽度的1%，
      2. 100vw视口宽度的100%，
        • 🪶即1vw=视口宽度1001vw=\frac{\text{视口宽度}}{100}1vw=100视口宽度​【常用】。
    2. vh(viewport height) ⥤ 视口高度的百分比，
      • 1vh视口高度的1%，
      • 100vh视口高度的100%
        • 🪶即1vh=视口高度1001vh=\frac{\text{视口高度}}{100}1vh=100视口高度​。
    3. vmin ⥤ 取当前vw和vh中较小的那一个值。
    4. vmax ⥤ 取当前vw和vh中较大的那一个值。

  • 🎭图示

  • 🔔提示

    • 👉100vw 等于 视口宽度，100vh 等于 视口高度。
      • 🍒例如
        • 浏览器宽度（视口宽度）为750px，高度（视口高度）为900px，
          • 则100vw=750px100vw=750px100vw=750px，100vh=900px100vh=900px100vh=900px，
          • 推1vw=750px100=7.5px1vw=\frac{750px }{100}=7.5px1vw=100750px​=7.5px，1vh=900px100=9px1vh=\frac{900px }{100}=9px1vh=100900px​=9px。

vw、vh 与 % 的区别是什么？

🔱前言

  • % 是相对于【父元素】的宽度或高度的百分比，而vw，vh是相对于【视口】的宽度或高度的百分比。

区别 ➾ 是否包含滚动条

  1. 没有滚动条的时候，
    • 100vw=100%
  2. 有滚动条的时候，
    1. 100vw包含滚动条。
    2. 100%不包含滚动条。

🎭图示

🔔提示

  • 当高度超出视口大小时，将出现滚动条。

视口单位优势

  • vh能够直接获取高度，
  • 而用%在没有设置body高度的情况下，无法正确获取可视区域的高度。

🔥案例

  • 让body元素与浏览器的视口一样高

vw布局

vw布局，🪶即将所有需要适配元素的属性都使用vw单位。

🧮计算公式

需要适配元素的属性大 ➾ vw单位

  • 需要适配元素的属性大小（vw）=设计稿测量的目标大小<宽高>（px）×100vw设计稿的宽（px）\text{需要适配元素的属性大小（vw）}=\frac{\text{设计稿测量的目标大小<宽高>（px）}\times{100vw}}{\text{设计稿的宽（px）}}需要适配元素的属性大小（vw）=设计稿的宽（px）设计稿测量的目标大小<宽高>（px）×100vw​
  • 🔔提示
    • vscode中的插件px2vw，可帮助将px数值转化为vw数值。

如何把100px换算成vw❓

根据设计稿的手机型号来进行换算。

如何判断UI给的二倍图的设计稿是什么手机型号呢？

🍒例如

  • 用ps之类的软件打开UI给的二倍图的设计稿，
    • 如果测量得到的是750px，除以2后就是 375px，
    • 🪶即该设计稿的机型是iPhone6。

现在我们知道的设计稿是什么机型，那就可以开始换算vw单位了。

  • 根据设计稿iphone5 ⥤ 320px=100vw，
    • 根元素最终设置为: html{ font-size: 31.25vw; }
  • 换算演示
    • 1px=100vw320px=0.3125vw1px=\frac{100vw}{320px}=0.3125vw1px=320px100vw​=0.3125vw
    • 进而100px=100px×100vw320px=31.25vw100px=100px\times{\frac{100vw}{320px}}=31.25vw100px=100px×320px100vw​=31.25vw这样就完成了，
      • 100px换算成，对应屏幕大小的单位为vw的数值。

🔥案例

      4. 边框为1px不缩放。

CSS3中的calc()

在CSS3中calc()函数，允许我们对属性值执行数学运算。

🔥案例

  • 使用calc()函数指定宽度值为两个或更多数值相加的结果，
    • 而不是把元素宽度值声明为一个静态像素值。
为什么要使用calc()？

📌原因

  • 使用calc()函数，我们可以组合不同的单位。
    • 具体来说，就是我们可以混合使用各种单位来进行计算，
      • 🍒例如:百分比、px、em、rem等单位都可以混在一起使用。

🔥案例

  • 创建一个表达式，它将从百分比值中减去像素值。

vw + rem布局

原理

由于rem实现时，我们需要借助Js，为解决这个问题，

  • 我们将rem布局原理中，当前视口宽用100vw表示，代替使用Js计算。

🔱前言

  • vw+remvw+remvw+rem布局是将根元素字体大小使用vw单位表示，而其他容器使用rem单位表示。

🧮计算公式

根元素字体大小 ➾ vw单位

  • 当前视口html字体大小（vw）=设计稿html的字体大小（px）×100vw设计稿的宽（px）\text{当前视口html字体大小（vw）}=\frac{\text{设计稿html的字体大小（px）}\times{100vw}}{\text{设计稿的宽（px）}}当前视口html字体大小（vw）=设计稿的宽（px）设计稿html的字体大小（px）×100vw​

🧮计算公式

其他容器属性大小 ➾ rem单位

  • 其他容器属性大小=[设计稿测量的目标大小<宽高>（px）根元素fontSize （px）]rem\text{其他容器属性大小}=[\frac{\text{设计稿测量的目标大小<宽高>（px）}}{\text{根元素fontSize （px）}}]rem其他容器属性大小=[根元素fontSize （px）设计稿测量的目标大小<宽高>（px）​]rem

适配方案对比

  1. 移动端布局方案
    1. 适配原理简单。
    2. 需要使用Js。
    3. 直接使用设计稿标注无需换算。
    4. 方案死板，只能实现页面级别整体缩放。
  2. 动态rem布局方案
    1. 适配原理稍复杂。
    2. 需要使用Js。
    3. 设计稿标注的px需要换算到CSS的rem ⥤ 计算简单。
    4. 方案灵活，既能实现整体缩放，又能实现局部不缩放。
  3. vw方案布局方案
    1. 适配原理简单。
    2. 不需要Js即可适配。
      • 🔔提示
        • 在移动端我们一般都可以认为，100vw就是屏幕宽度。
          • 若使用vw布局，就不需要再像rem那样，在Js中去动态设置根元素的font-size。
    3. 设计稿标注的px需要换算到CSS的vw计算复杂。
    4. 方案灵活，既能实现整体缩放，又能实现局部不缩放。
  4. vw + rem布局【优先选择】
    1. 适配原理简单。
    2. 不需要Js即可适配。
    3. html字体大小需要换算到CSS的vw计算复杂
    4. 设计稿标注的px需要换算到CSS的rem计算复杂。
    5. 方案灵活，既能实现整体缩放，又能实现局部不缩放。
