011.瀑布流多列布局案例

糖丸2023-05-302023-12-29

多列布局案例-瀑布流

设计稿

使用多列布局时出现的问题❓

  • 在底部出现内容被截断，然后剩余的部分，出现在下一列的头部的位置。

📌原因

  • 在Css3规范里，各列的高度是均衡的 （column-fill）。
    • 浏览器会自动调整每列会填充多少文本，然后让整个布局的高度保持均衡一致。
  • 而我们在做这种瀑布流的类似布局的时候，
    • 往往只需要设置每个父容器的宽度，然后子元素高度通过内容填充来自适应。
  • 所以，浏览器在处理的时候，为了保持高度的均衡分配，会将我们的布局截断，然后造成布局错乱。

🎭图示

🛠️解决方法

  • 给子容器加上[page-]break-inside: avoid;

❗注意

  • 是子容器（容纳图片的容器）不是父容器。

Css page-break-inside 属性

  • ✨作用

    • 用于设置是否在指定元素中插入分页符。

  • ❗注意

    1. 您不能对绝对定位的元素使用此属性。
    2. 请尽可能少地使用分页属性，并且避免在表格、浮动元素、带有边框的块元素中使用分页属性。
默认值auto
继承no
JavaScript 语法object.style.pageBreakInside=“avoid”
  • 属性值
值描述
auto如果必要则在元素内部插入分页符。
avoid避免在元素内部插入分页符。

Css break-inside 属性

  • ✨作用

    • 规定在指定元素内部是否应发生分页（page-break）、分列（column-break）或分区（region-break）。

  • 🔔提示

    • break-inside 属性扩展了 Css2 的 page-break-inside 属性。
      • 通过使用 break-inside，您可以告知浏览器在图像、代码片段、表格以及列表内部避免中断。

  • 🎯特性

默认值auto
继承否
JavaScript 语法object.style.breakInside=“avoid”
  • 属性值
值描述
auto在元素内自动进行分页、分列、分区。
avoid避免在元素内出现页、列、区域中断。
avoid-page避免在元素内分页。
avoid-column避免在元素内分列。
avoid-region避免在元素内分区。
分页别名
  • 出于兼容性原因，浏览器应将旧版page-break-inside属性视为break-inside 的别名。
    • 这样可确保使用page-break-inside 网站继续按设计工作。
