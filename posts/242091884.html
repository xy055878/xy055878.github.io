007.弹性盒

弹性盒

  • 🔱前言

    • 弹性布局是一种新的布局方式，
      • 特别适合移动端布局。

  • 💎描述

    • CSS3 弹性盒是一种
      • 当页面需要适应不同的屏幕大小以及设备类型时，
        • 确保元素拥有恰当的行为的布局方式。

  • 引入弹性盒布局模型的目的，

    • 是提供一种更加有效的方式，
      • 来对一个容器（父元素）中的
        • 子元素进行排列。

🔔提示

  • 弹性布局
    • 即可以应用于容器父级元素，
    • 也可以应用于行内元素。

CSS3 弹性盒子内容

  • 弹性盒子由弹性容器【父元素】和 弹性子元素组成。
  • 启用弹性盒
    • 弹性容器通过设置 display 属性的值为
    • flex 或 inline-flex将元素定义为弹性容器。
  • 弹性容器内包含了一个或多个弹性子元素。

❗注意

  • 弹性容器外及弹性子元素内是正常渲染的。
    • 弹性盒子只定义了弹性子元素
    • 如何在弹性容器内布局。
  • 也就是说，弹性盒子的设置只对自己有效，
    • 对其中的子元素和外部元素无效。

弹性盒模型

  • 🔱前言

    • 布局的传统解决方案
      • 基于盒模型，依赖display + position + float，
        • 它对于那些特殊的布局非常不方便。
        • 🍒例如
          • 垂直居中就不容易实现。
  • 启用弹性盒

🔉说明

  • flex：将元素作为弹性伸缩盒显示。
  • inline-flex：将元素作为内联块级弹性伸缩盒显示。

  • ⏳详述

    • 当弹性容器没有设置宽度大小限制时，
      1. 当display指定为flex时，
        • 弹性容器的宽度
          • 会填充父容器 。
      2. 当display指定为inline-flex时，
        • 弹性容器的宽度
          • 会包裹子元素。
  • 🔥案例

  • display: flex;

  • display: inline-flex;

  • ⏳详述

    • 当弹性容器有设置宽度大小限制时，
      1. display指定为flex。
        • 等价于
      2. display指定为inline-flex。

  • 🔥案例

    • display: flex;＆display: inline-flex;

弹性容器属性

①flex-direction 属性

  • ✨作用

    • 指定弹性子元素，
      • 在弹性容器中的位置。

    • ⏳详述

      • 修改主轴和侧轴的方向

    • 🎯特性

默认值 row
继承 否
JavaScript 语法 object.style.flexDirection=”column-reverse”
  • 语法
  • 属性值
值 描述
row 以第一个弹性子元素为基准
弹性子元素和 main start（左） & cross start（上）对齐
左对齐
row-reverse 以第一个弹性子元素为基准
弹性子元素和 main end（右）& cross start（上）对齐
右对齐
column 以第一个弹性子元素为基准
弹性子元素和 cross start（左）& main start（上）对齐
上对齐
column-reverse 以第一个弹性子元素为基准
弹性子元素和 cross start（左）& main end（下）对齐
下对齐
  • 🔥案例
    • flex-direction: row;

  • 🔥案例

    • flex-direction: row-reverse;

🔔提示

  • 如果我们设置 direction 属性
    • 为 rtl (right-to-left)，
      • 弹性子元素的排列方式也会改变，
      • 页面布局也跟着改变。
  • 🔥案例
    • direction: rtl;

  • 🔥案例
    • flex-direction: column;

  • 🔥案例
    • direction: rtl; + flex-direction: column;

  • 🔥案例
    • flex-direction: column-reverse;

②CSS flex-wrap 属性

  • ✨作用
    • 指定弹性子元素，
      • 在弹性容器中的换行方式。
  • ⏳详述
    • 使弹性子元素自动换行（换行）。
  • 🎯特性
默认值 nowrap
继承 否
JavaScript 语法 object.style.flexWrap=”nowrap”
  • 语法
  • 属性值
值 描述
nowrap 弹性容器为单行,
🪶即弹性子元素始终在同一行。

①当弹性子元素数量有很多时，
弹性容器将不断压缩
弹性子元素的content width，

②当弹性子元素的content width无法压缩时，
弹性子元素将进行同行溢出。
wrap 弹性容器为多行,
🪶即弹性子元素可能将在不同行。

当弹性子元素数量有很多时，
弹性子元素溢出本行的
将会被放置到新行。
wrap-reverse 反转wrap排列，
🪶即排列顺序和wrap相反。
  • 🔥案例
    • flex-wrap: nowrap;

  • 🔥案例
    • 当弹性子元素数量有很多时，
      • 弹性容器将不断压缩，弹性子元素的content width。

  • 👻特别的
    • 同行溢出

  • 🔥案例
    • flex-wrap: wrap;

👁️‍🗨️分析

  • 等间距出现的原因
    • align-content: stretch;
  • 🔥案例
    • flex-wrap: wrap;

👁️‍🗨️分析

  • ①当内容超出div<item5>的宽度时，
    • 内容在同一行。
  • ②等间距不出现
    • 🪶即align-content: stretch;失效。
  • 原因
    • 弹性容器未固定高度，
      • 故无法计算元素间距。
  • 问题❓
    • ①当内容超出div<item5>的宽度时，
      • 内容在同一行。

🔱前言

  • 换行符

    • 连字符『-』、中英文问号『?』、中文『：』
    • 英文空格『空格』、『CJK字符』、Emoji『😊』
    • 🔔提示
      • CJK文本指中国、日本、韩国的文字。

  • 浏览器默认的换行规则

    • ①英文中如果存在换行符，
    • ②行内宽度不足，
      • 则浏览器会在换行符后自动换行。

CSS3 word-break 属性

  • ✨作用
    • 指定非CJK文本的断行规则。
  • 🎯特性
默认值 normal
继承 yes
JavaScript 语法 object.style.wordBreak=”break-all”
  • 语法
  • 属性值
值 描述
normal 使用浏览器默认的换行规则。
break-all 强行换行，
🪶即碰到边界一律换行。
keep-all 只能在换行符处<不包括CJK字符>换行。

❗注意

  • word-break : break-all;
    • 所有浏览器都支持。
  • 移动端还不适合使用<兼容性差>
    • word-break : keep-all;
  • 🔥案例
    • word-break：break-all;

👁️‍🗨️分析

  • 文本都不会溢出。

🪄总结

  • word-break: break-all;的作用是
    • 所有的都换行，很霸道的那种，毫不留情，
    • 一点儿空隙都不放过的那种换行。

🔥案例

  • word-break：keep-all;

👁️‍🗨️分析

  • 由上图中，所有字符达到边界时，
    • 很明显只有换行符处
      • 换行显示。

🪄总结

  • word-break: keep-all;的作用是
    • 只有换行符处换行。

CSS3 word-wrap 属性

  • ✨作用
    • 指定长单词或URL地址的换行规则。
  • 🎯特性
默认值 normal
继承 yes
JavaScript 语法 object.style.wordWrap=”break-word”
  • 语法
  • 属性值
值 描述
normal 使用浏览器默认的换行规则。
break-word 允许长单词或URL地址进行换行。
💦另，一行单词中，
实在没有其他靠谱的换行符font>的时候，
才进行换行。

👻特别的

  • 由于word-break和word-wrap长得比较像，
    • 而且属性值也有类似之处。
  • 所以在CSS3规范里，这个属性的名称被修改了，
    • 叫作 overflow-wrap。
  • 虽然这个新属性名称修改了，
    • 显得语义更准确，也更容易区别和记忆。
  • 但是只有Chrome和 Safari等
    • WebKit内核的浏览器支持这个新属性。
  • 因此，虽然换了个好看好用的新名字，
    • 但是为了兼容性，目前还是乖乖地使用word-wrap 吧。
  • 🔥案例
    • word-wrap: break-word;

👁️‍🗨️分析

  • 文本都不会溢出。

🪄总结

  • word-wrap: break-word;的作用是
    • ①如果这一行文字有换行符,
      • 则换行。
    • ②当一个单词，
      • 一整行都显示不下时，
        • 则换行。
  • 解决方案

  • 🔥案例
    • flex-wrap: wrap-reverse;

  • 🔥案例
    • flex-direction: column; +flex-wrap: wrap;
