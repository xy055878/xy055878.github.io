弹性盒

弹性布局是一种新的布局方式，特别适合移动端布局。

  • 💎描述

    • CSS3 弹性盒是一种当页面需要适应不同的屏幕大小以及设备类型时，
      • 确保元素拥有恰当的行为的布局方式。

引入弹性盒布局模型的目的

  • 提供一种更加有效的方式，来对一个容器（父元素）中的子元素进行排列。

🔔提示

  • 弹性布局即可以应用于容器父级元素，也可以应用于行内元素。

CSS3 弹性盒的组成

弹性盒子由弹性容器【父元素】和 弹性子元素组成。

  • 🔔提示
    • 弹性容器内包含了一个或多个弹性子元素。

❗注意

  • 弹性容器外及弹性子元素内是正常渲染的。
    • 弹性盒子只定义了弹性子元素，如何在弹性容器内布局。
  • 也就是说，弹性容器的设置只对自己有效，对其中的子元素和外部元素无效。

🔔提示

  • 具有flex环境的父元素，通常有一条主轴和一条侧轴。

    • 默认情况下，主轴就是水平从左向右，侧轴是垂直从上到下。

弹性盒模型

🔱前言

  • 布局的传统解决方案
    • 基于盒模型，依赖display + position + float，它对于那些特殊的布局非常不方便。
      • 🍒例如
        • 垂直居中就不容易实现。

启用弹性盒

弹性容器通过设置 display 属性的值为

  • flex 或 inline-flex将元素定义为弹性容器。

🔉说明

  1. flex: 将元素作为弹性伸缩盒显示。
  2. inline-flex: 将元素作为内联块级弹性伸缩盒显示。

  • ⏳详述

    • 当弹性容器没有设置宽度大小限制时，
      1. 当display指定为flex时，
        • 弹性容器的宽度<继承于父元素的>会填充弹性盒 。
      2. 当display指定为inline-flex时，
        • 弹性容器的宽度会包裹子元素。

  • 🔥案例

  • display: flex;

  • 👁️‍🗨️分析

    • 当弹性容器没有设置宽度大小限制，此时相对于父元素（body）。

  • display: inline-flex;

  • ⏳详述

    • 当弹性容器有设置宽度大小限制时，
      • display: flex;等价于 display: inline-flex;

②CSS flex-wrap 属性
  • ✨作用
    • 指定弹性子元素，在弹性容器中的换行方式。
    • ⏳详述
      • 使弹性子元素自动换行（折行）。
  • 🎯特性
默认值nowrap
继承否
JavaScript 语法object.style.flexWrap=“nowrap”
  • 语法
  • 属性值
值描述
nowrap弹性容器为单行,🪶即弹性子元素始终在同一行。

①当弹性子元素数量有很多时，
弹性容器将不断压缩弹性子元素的content width，

②当弹性子元素的content width无法压缩时，
弹性子元素将进行同行溢出。
wrap弹性容器为多行,🪶即弹性子元素可能将在不同行。

当弹性子元素数量有很多时，
弹性子元素溢出本行的将会被放置到新行。
wrap-reverse反转wrap排列，🪶即排列顺序和wrap相反。
  • 🔥案例
    • flex-wrap: nowrap;

  • 🔥案例
    • 当弹性子元素数量有很多时，弹性容器将不断压缩，弹性子元素的content width。

  • 👻特别的
    • 同行溢出

  • 🔥案例
    • flex-wrap: wrap;

👁️‍🗨️分析

  • 等间距出现的📌原因
    • align-content: stretch;
  • 🔥案例
    • flex-wrap: wrap;

问题❓

  1. 当内容超出div<item5>的宽度时，内容在同一行。
  2. 等间距不出现
    • 🪶即align-content: stretch;失效。

📌原因

  • 弹性容器未固定高度，故无法计算元素间距。

常用换行符

  1. 连字符『-』、中英文问号『?』、中文『: 』
  2. 英文空格『空格』、『CJK字符』、Emoji『😊』

🔔提示

  • CJK文本指中国、日本、韩国的文字。

浏览器默认的换行规则

英文中如果存在换行符，行内宽度不足，

  • 则浏览器会在换行符后自动换行。

CSS word-wrap 属性

  • ✨作用

    • 指定长单词或URL地址的换行规则。

  • 🎯特性

默认值normal
继承yes
JavaScript 语法object.style.wordWrap=“break-word”
  • 语法
  • 属性值
值描述
normal使用浏览器默认的换行规则。
break-word允许长单词或URL地址进行换行。
🪁另，一行单词中，实在没有其他靠谱的换行符font>的时候，才进行换行。

👻特别的

  • 由于word-break和word-wrap长得比较像，而且属性值也有类似之处。
    • 所以在CSS3规范里，这个属性的名称被修改了，叫作 overflow-wrap。
  • 虽然这个新属性名称修改了，显得语义更准确，也更容易区别和记忆。
    • 但是只有Chrome和 Safari等，WebKit内核的浏览器支持这个新属性。
  • 因此，虽然换了个好看好用的新名字，
    • 但是为了兼容性，目前还是乖乖地使用word-wrap 吧。
  • 🔥案例
    • word-wrap: break-word;

👁️‍🗨️分析

  • 文本都不会溢出。

🪄总结

  • word-wrap: break-word;的作用是
    1. 如果这一行文字有换行符，则换行。
    2. 当一个单词，一整行都显示不下时，则换行。
  • 🛠️解决方法

  • 🔥案例
    • flex-wrap: wrap-reverse;

  • 🔥案例
    • flex-direction: column; +flex-wrap: wrap;

默认值0

  • 语法
  • 属性值
值描述
number默认值是 0。
一个非负数，指定弹性子元素，
在主轴<main axis>方向上的扩展比例。
剩余空间
  • 🔉说明
    • 父元素在主轴方向上的可用空间。
  • 🔔提示
    • 具有flex环境的父元素，
      • 通常有一条主轴和一条侧轴，
      • 默认情况下，主轴就是水平从左向右，侧轴是垂直从上到下。
  • 🧮计算公式
    • 剩余空间=父元素宽度−所有子元素宽度和\text{剩余空间}=\text{父元素宽度}-\text{所有子元素宽度和}剩余空间=父元素宽度−所有子元素宽度和

具体分析

  • ①flex-grow: 0;<默认值>

    • 表示该弹性子元素正常显示宽度，针对弹性容器 剩余空间 不进行处理。

  • 🔥案例

  • ②flex-grow: 1 / 2 / ...;

    • 只给 一个弹性子元素
      • 添加flex-grow属性时，
        • 则将该行 剩余空间， 全部分给该弹性子元素。
    • 🍒例如
      • 只给文本2设置flex-grow: 1;

  • 🔥案例

👁️‍🗨️分析

  • 文本2将该行 剩余空间 占满，此时所有弹性子元素的总宽度和弹性容器相等。

  • ③同时给多个弹性子元素，添加flex-grow属性时，

    • 将该行 剩余空间 除以flex-grow属性值的总和，再按份数< flex-grow属性值 >分给盒子弹性子元素。
    • 🍒例如
      • 给文本2添加flex-grow: 1;
      • 又给文本3设置了flex-grow: 2;

      • 🪶即文本2占1份，文本3占2份。

④CSS flex-shrink 属性

  • ✨作用

    • 用于指定弹性子元素，在主轴<main axis>方向上的收缩比例。
      • 🪶即当弹性子元素总宽度超过弹性容器的宽度时，对超出的宽度<溢出总量>进行处理。

  • 🎯特性

默认值1
继承否
JavaScript 语法object.style.flexShrink=“5”
  • 语法
  • 属性值
值描述
number默认值是 1。
一个非负数，指定弹性子元素，
在主轴<main axis>方向上的收缩比例。

❗注意

  • 弹性子元素仅在其宽度之和大于刚刚弹性容器的时候才会发生收缩。
      • 其收缩的大小是依据 flex-shrink 的值。
具体分析
  • ① flex-shrink: 1; <默认值>
    • 当子级元素总宽度超出弹性容器的宽度时，会自动进行挤压。
  • 🍒例如
    • 超出部分被平均分配给四个弹性子元素，弹性子元素内部进行了挤压。

  • ② flex-shrink: 0;
    • 浏览器对弹性子元素不进行挤压，该元素正常显示大小。
      • 🪶即将超出空间分配给其他弹性子元素挤压。
    • 🍒例如
      • 给文本2设置flex-shrink: 0；

  • ③给多个弹性子元素添加flex-shrink属性时

    • 🔥案例
      • flex-direction: row;

👁️‍🗨️分析

  • ①如果弹性子元素没有显示定义fiex-shrink属性，

    • 那么弹性子元素将会自动按照的默认值1，来计算收缩比例。

  • ②本例中A、B、C显式定义了flex-shrink为1，D、E定义了 flex-shrink为2，

    • 所以计算出来总共将剩余空间分成了7份，
      • 其中A、B、C占1份，D、E占2份，即1:1:1:2:2。

  • 我们可以看到弹性容器定义为500px，

    • 弹性子元素被定义为120px，弹性子元素相加之后为600px

⑤CSS flex-basis 属性

  • ✨作用

    • 用于指定弹性子元素在主轴<main axis>方向上的初始宽度。

  • ❗注意

    1. 此初始宽度又称为伸缩基准值。
    2. 如果主轴方向改变则flex-basis为初始高度。

  • 🎯特性

默认值auto
继承否
JavaScript 语法object.style.flexBasis=“200px”
  • 语法
  • 属性值

number一个单位或者一个百分比。
用于指定弹性子元素，
在主轴<main axis>方向上的初始宽度。
auto初始宽度等于弹性子元素的宽度。
如果弹性子元素未指定宽度，则初始宽度将由内容决定。

    3. flex-basis: 120px;
      • 表示设置弹性子元素的初始宽度为120px。

CSS flex 属性

  • 🔉说明

    • flex 属性是 flex-grow、flex-shrink、flex-basis 属性的简写属性。

  • 🎯特性

默认值0 1 auto
继承否
JavaScript 语法object.style.flex=“1”
  • 语法
  • 属性值
值描述
flex-grow一个非负数，
用于指定弹性子元素，在主轴
方向上的扩展比例。
flex-shrink一个非负数，
用于指定弹性子元素，在主轴< main axis >方向上的收缩比例。
flex-basis用于指定弹性子元素，在主轴< main axis >方向上的初始宽度。
auto与 1 1 auto 相同。
none与 0 0 auto 相同。
initial设置该属性为它的默认值，即为 0 1 auto。请参阅inherit。
具体分析

  • ①flex: auto;

    • 🪶即flex: 1 1 auto;

  • ✨作用

    • 根据主轴自动伸缩，以占用所有剩余空间。

    • ⏳详述

      • 放大且缩小。

  • ②flex: none;

    • 🪶即flex: 0 0 auto;

  • ✨作用

    • 在任何情况都不会发生伸缩。

    • ⏳详述

      • 即不放大也不缩小。

  • ③flex: initial;

    • 🪶即flex: 0 1 auto; < 默认值 >

  • ✨作用

    1. 弹性子元素总宽度小于弹性盒宽度时，对剩余空间不进行处理。
    2. 弹性子元素总宽度大于弹性盒宽度时，对弹性子元素进行挤压。
      • ⏳详述
        • 不放大但缩小。

  • ④一个非负数字:

    • 🪶即flex: 非负数字 1 0%;

    • 🔉说明

      1. 该非负数字为flex-grow值，
      2. flex-shrink取1，
      3. flex-basis取0%。

    • 👻特别的

      • flex: 1;
        • 🪶即flex: 1 1 0%;

    • 🔔提示

      • 一般弹性盒中只设置一个弹性元素为flex: 1;。

  • ⑤一个长度或百分比

    • 🪶即flex: 1 1 一个长度或百分比;
    • 🔉说明
      1. 一个长度或百分比视为flex-basis值，
        • ❗注意
          • 0% 是一个百分比而不是一个非负数字。
      2. flex-grow取1 < 注意 > ，
      3. flex-shrink取1。

  • 🔥案例

  • 🔥案例

  • ⑥两个非负数字
    • 🪶即flex: 非负数字① 非负数字② 0%;
    • 🔉说明
      1. 两个非负数字分别视为 flex-grow 和flex-shrink的值，
      2. flex-basis 取 0%。
  • ⑦一个非负数字，一个长度或百分比
    • 🪶即flex: 非负数字 1 一个长度或百分比;
    • 🔉说明
      1. 非负数字视为 flex-grow 的值，
      2. 长度或百分比视为flex-basis 的值，
      3. flex-shrink 取 1。
综合练习

🎏需求

  • 设置父元素 width = 600px;，

    • 三个子元素A、B、C，

      1. A的width为140px，flex为2 1 0%，
      2. B的width为100px，flex为2 1 auto，
      3. C不设width，flex为1 1 200px。

    • 问题❓

      • 求A、B、C三个元素的width是多少？

  • 计算过程

    • 主轴上的弹性盒总尺寸为600px，

      1. **子元素的总基准值: **0% + auto + 200px = 300px。

        • 🔔提示

          • 0%即0宽度，auto对应取主轴尺寸< main size>

          • 🪶即100px（B元素的宽度）。

      2. **扩展比例之和: ** 2 + 2 + 1 = 5。

**剩余空间: **600px - 300px = 300px

  • 剩余空间分配

    1. A和B各分配 2/5，🪶即各得120px；
    2. C分配 1/5，得60px。

  • 最终各个元素的宽度分别为:

  1. A的宽度 = 0%（0px） + 120px = 120px

    • 🔔提示

      • 当A的基准值取0%时，是把该元素当作零尺寸的，

      • 所以即使声明width为140px，也没有用。

  2. B的宽度 = auto（100px） + 120px = 220px

    • 🔔提示

      • 当B的基准值取auto时，根据规则基准值使用值是主轴尺寸< main size>值，

      • 🪶即 100px（B的width），所以这100px不会纳入剩余空间。

  3. C的宽度 = 200px + 60px = 260px
