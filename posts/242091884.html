弹性盒

  • 🔱前言

  • 弹性布局是一种新的布局方式，

    • 特别适合移动端布局。

  • 💎描述

    • CSS3 弹性盒是一种
      • 当页面需要适应不同的屏幕大小以及设备类型时，
        • 确保元素拥有恰当的行为的布局方式。

  • 引入弹性盒布局模型的目的，

    • 是提供一种更加有效的方式，
      • 来对一个容器（父元素）中的
        • 子元素进行排列。

🔔提示

  • 弹性布局
    • 即可以应用于容器父级元素，
    • 也可以应用于行内元素。

CSS3 弹性盒的组成

  • 弹性盒子由弹性容器【父元素】和 弹性子元素组成。
  • 启用弹性盒
    • 弹性容器通过设置 display 属性的值为
      • flex 或 inline-flex将元素定义为弹性容器。
  • 弹性容器内包含了一个或多个弹性子元素。

❗注意

  • 弹性容器外及弹性子元素内是正常渲染的。
    • 弹性盒子只定义了弹性子元素，
      • 如何在弹性容器内布局。
  • 也就是说，弹性容器的设置只对自己有效，
    • 对其中的子元素和外部元素无效。

  • 🔔提示

  • 具有flex环境的父元素，

    • 通常有一条主轴和一条侧轴，

    • 默认情况下，

      • 主轴就是水平从左向右，侧轴是垂直从上到下。

弹性盒模型

  • 🔱前言

    • 布局的传统解决方案
      • 基于盒模型，依赖display + position + float，
        • 它对于那些特殊的布局非常不方便。
        • 🍒例如
          • 垂直居中就不容易实现。

启用弹性盒

🔉说明

  • flex：将元素作为弹性伸缩盒显示。
  • inline-flex：将元素作为内联块级弹性伸缩盒显示。

  • ⏳详述

    • 当弹性容器没有设置宽度大小限制时，
      1. 当display指定为flex时，
        • 弹性容器的宽度
          • 会填充弹性盒 。
      2. 当display指定为inline-flex时，
        • 弹性容器的宽度
          • 会包裹子元素。
  • 🔥案例
  • display: flex;

  • display: inline-flex;

  • ⏳详述

    • 当弹性容器有设置宽度大小限制时，
      1. display指定为flex。
        • 等价于
      2. display指定为inline-flex。

  • 🔥案例

    • display: flex;＆ display: inline-flex;

CSS word-wrap 属性

  • ✨作用
    • 指定长单词或URL地址的换行规则。
  • 🎯特性
默认值normal
继承yes
JavaScript 语法object.style.wordWrap="break-word"
  • 语法
  • 属性值
值描述
normal使用浏览器默认的换行规则。
break-word允许长单词或URL地址进行换行。

💦另，一行单词中，
实在没有其他靠谱的换行符font>的时候，
才进行换行。

👻特别的

  • 由于word-break和word-wrap长得比较像，
    • 而且属性值也有类似之处。
  • 所以在CSS3规范里，这个属性的名称被修改了，
    • 叫作 overflow-wrap。
  • 虽然这个新属性名称修改了，
    • 显得语义更准确，也更容易区别和记忆。
  • 但是只有Chrome和 Safari等
    • WebKit内核的浏览器支持这个新属性。
  • 因此，虽然换了个好看好用的新名字，
    • 但是为了兼容性，目前还是乖乖地使用word-wrap 吧。
  • 🔥案例
    • word-wrap: break-word;

👁️‍🗨️分析

  • 文本都不会溢出。

🪄总结

  • word-wrap: break-word;的作用是
    • ①如果这一行文字有换行符,
      • 则换行。
    • ②当一个单词，
      • 一整行都显示不下时，
        • 则换行。
  • 解决方案

  • 🔥案例
    • flex-wrap: wrap-reverse;

  • 🔥案例
    • flex-direction: column; +flex-wrap: wrap;

③CSS flex-grow 属性

  • ✨作用

    • 用于指定弹性子元素，
      • 在主轴<main axis>方向上的扩展比例。

  • 🎯特性

默认值0
继承否
JavaScript 语法object.style.flexGrow="5"
  • 语法
  • 属性值
值描述
number默认值是 0。

一个非负数，指定弹性子元素，
在主轴<main axis>方向上的扩展比例。
剩余空间
  • 🔉说明
    • 父元素在主轴方向上的可用空间。
  • 🔔提示
    • 具有flex环境的父元素，
      • 通常有一条主轴和一条侧轴，
      • 默认情况下，
        • 主轴就是水平从左向右，侧轴是垂直从上到下。
  • 公式
    • 剩余空间 = 父元素宽度 - 所有子元素宽度和
  • 🎭图示

具体分析

  • ①flex-grow: 0;<默认值>

    • 表示该弹性子元素正常显示宽度，
      • 针对弹性容器 剩余空间 不进行处理。

  • 🔥案例

  • ②flex-grow: 1 / 2 / ...;

    • 只给 一个弹性子元素
      • 添加flex-grow属性时，
        • 则将该行 剩余空间
          • 全部分给该弹性子元素。
    • 🍒例如
      • 只给文本2设置flex-grow：1;

  • 🔥案例

👁️‍🗨️分析

  • 文本2将该行 剩余空间 占满，
    • 此时所有弹性子元素的总宽度，
      • 和弹性容器相等。

  • ③同时给多个弹性子元素，添加flex-grow属性时

      • 将该行 剩余空间
        • 除以flex-grow属性值的总和，
          • 再按份数<flex-grow属性值>分给盒子弹性子元素。
    • 🍒例如
      • 给文本2添加flex-grow：1;
      • 又给文本3设置了flex-grow：2;
        • 则空间基数的计算方法为： 剩余空间 /（1+2），
        • 文本2占1份，文本3占2份。

  • 🔥案例

④CSS flex-shrink 属性

  • ✨作用

    • 用于指定弹性子元素，

      • 在主轴<main axis>方向上的收缩比例。

    • 🪶即当弹性子元素总宽度超过弹性容器的宽度时，

      • 对超出的宽度<溢出总量>进行处理。

  • 🎯特性

默认值1
继承否
JavaScript 语法object.style.flexShrink="5"
  • 语法
  • 属性值
值描述
number默认值是 1。
一个非负数，指定弹性子元素，
在主轴<main axis>方向上的收缩比例。

❗注意

  • 弹性子元素仅在其宽度之和
    • 大于弹性容器的时候才会发生收缩。
      • 其收缩的大小是依据 flex-shrink 的值。
具体分析
  • ① flex-shrink: 1; <默认值>
  • 当子级元素总宽度超出弹性容器的宽度时，
    • 会自动进行挤压。
  • 🍒例如
    • 超出部分被平均分配给四个弹性子元素，
      • 弹性子元素内部进行了挤压。

  • ② flex-shrink: 0;
    • 浏览器对弹性子元素不进行挤压，
      • 该元素正常显示大小。
    • 将超出空间分配给
      • 其他弹性子元素挤压。
    • 🍒例如
    • 给文本2设置flex-shrink：0；

  • ③给多个弹性子元素添加flex-shrink属性时
  • 🔥案例
    • flex-direction: row;

👁️‍🗨️分析

  • ①如果弹性子元素没有
    • 显示定义fiex-shrink属性，
  • 那么弹性子元素将会自动按照的默认值1，
    • 来计算收缩比例。
  • ②本例中A、B、C显式定义了flex-shrink为1，
    • D、E定义了 flex-shrink为2，
  • 所以计算出来总共将剩余空间分成了7份，
    • 其中A、B、C占1份，D、E占2份，即1 : 1 : 1 : 2 : 2。

  • 我们可以看到弹性容器定义为500px，

    • 弹性子元素被定义为120px，
    • 弹性子元素相加之后为600px
      • 🪶即元素的总宽度，
    • 超出弹性容器100px。
      • 总溢出量 = 元素的总宽度 - 弹性容器宽度

  • 所以可得出加权综合空间，

    • 100 * 1 + 100 * 1 + 100 * 1 + 100 * 2 + 100 * 2 = 700px。
  • 于是我们可以计算A、B、C、D、E
    • 将被移除的溢出量是多少。
  • 🔔提示
计算被移除溢出量的公式

  • 🍒例如
  • E被移除溢出量：（100 * 2 / 700）* 100），即约等于28px。
  • 通俗的讲就是
  • 将空间分成n份< fiex-shrink属性值的总和 >
    • 则当前元素被移除溢出量为

  • 🍒例如

    • E被移除溢出量：2 / 7* 100，即约等于28px。

  • 🔥案例

    • flex-direction: column;

CSS flex 属性

  • 🔉说明

    • flex 属性是 flex-grow、flex-shrink、flex-basis 属性的简写属性。

  • 🎯特性

默认值0 1 auto
继承否
JavaScript 语法object.style.flex="1"
  • 语法
  • 属性值
值描述
flex-grow一个非负数，
用于指定弹性子元素，
在主轴
方向上的扩展比例。
flex-shrink一个非负数，
用于指定弹性子元素，
在主轴
方向上的收缩比例。
flex-basis用于指定弹性子元素
在主轴
方向上的初始宽度。
auto与 1 1 auto 相同。
none与 0 0 auto 相同。
initial设置该属性为它的默认值，即为 0 1 auto。
请参阅inherit。
具体分析
  • ①flex: auto;
    • 🪶即flex: 1 1 auto;
    • ✨作用
      • 根据主轴自动伸缩，以占用所有剩余空间。
    • ⏳详述
      • 放大且缩小。
  • ②flex: none;
    • 🪶即flex: 0 0 auto;
    • ✨作用
      • 在任何情况都不会发生伸缩。
    • ⏳详述
      • 即不放大也不缩小。

  • ③flex: initial;

    • 🪶即flex: 0 1 auto; < 默认值 >

    • ✨作用

      1. 弹性子元素总宽度小于弹性盒宽度时，
        • 对剩余空间不进行处理；
      2. 弹性子元素总宽度大于弹性盒宽度时，
        • 对弹性子元素进行挤压。

    • ⏳详述

      • 不放大但缩小。

  • ④一个非负数字：

    • 🪶即flex: 非负数字 1 0%;

    • 🔉说明

      1. 该非负数字为flex-grow值，
      2. flex-shrink取1，
      3. flex-basis取0%。

    • 👻特别的

      • flex: 1;
      • 🪶即flex: 1 1 0%;

    • 🔔提示

      • 一般弹性盒中只设置一个弹性元素为flex：1;。

  • ⑤一个长度或百分比

    • 🪶即flex: 1 1 一个长度或百分比;
    • 🔉说明
      1. 一个长度或百分比视为flex-basis值，
        • ❗注意
          • 0% 是一个百分比而不是一个非负数字。
      2. flex-grow取1 < 注意 > ，
      3. flex-shrink取1。

  • ⑥两个非负数字

    • 🪶即flex: 非负数字① 非负数字② 0%;
    • 🔉说明
      1. 两个非负数字
        • 分别视为 flex-grow 和flex-shrink的值，
      2. flex-basis 取 0%。

  • ⑦一个非负数字，一个长度或百分比

    • 🪶即flex: 非负数字 1 一个长度或百分比;
    • 🔉说明
      1. 非负数字视为 flex-grow 的值，
      2. 长度或百分比视为flex-basis 的值，
      3. flex-shrink 取 1。

  • 🔥案例

设置父元素 width = 600px;，

  • 三个子元素A、B、C，

    1. A的width为140px，flex为2 1 0%，
    2. B的width为100px，flex为2 1 auto，
    3. C不设width，flex为1 1 200px。

  • 问题❓

    • 求A、B、C三个元素的width是多少？

  • 计算过程

    • 主轴上的弹性盒总尺寸为600px，

      • 子元素的总基准值

        • 0% + auto + 200px = 300px，

        • 🔔提示

          • 0%即0宽度，

          • auto对应取主轴尺寸< main size>

            • 即100px（B元素的宽度）。
      • 扩展比例之和
        • 2 + 2 + 1 = 5，

  • 剩余空间

    • 600px - 300px = 300px ，
    • 剩余空间分配
      • A和B各分配 2/5，
        • 🪶即各得120px；
      • C分配 1/5，
        • 得60px。

  • 最终各个元素的宽度分别为

    • A的宽度 = 0%（0px） + 120px = 120px

      • 🔔提示
        • 当A的基准值取0%时，
        • 是把该元素当作零尺寸的，
        • 所以即使声明width为140px，也没有用。

    • B的宽度 = auto（100px） + 120px = 220px

      • 🔔提示
        • 当B的基准值取auto时，
        • 根据规则基准值使用值是主轴尺寸< main size>值，
        • 🪶即即 100px（B的width），
          • 所以这100px不会纳入剩余空间。

    • C的宽度 = 200px + 60px = 260px
