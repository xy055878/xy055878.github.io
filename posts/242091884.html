弹性盒

  • 🔱前言

  • 弹性布局是一种新的布局方式，

    • 特别适合移动端布局。

  • 💎描述

    • CSS3 弹性盒是一种
      • 当页面需要适应不同的屏幕大小以及设备类型时，
        • 确保元素拥有恰当的行为的布局方式。

  • 引入弹性盒布局模型的目的，

    • 是提供一种更加有效的方式，
      • 来对一个容器（父元素）中的
        • 子元素进行排列。

🔔提示

  • 弹性布局
    • 即可以应用于容器父级元素，
    • 也可以应用于行内元素。

CSS3 弹性盒的组成

  • 弹性盒子由弹性容器【父元素】和 弹性子元素组成。
  • 启用弹性盒
    • 弹性容器通过设置 display 属性的值为
      • flex 或 inline-flex将元素定义为弹性容器。
  • 弹性容器内包含了一个或多个弹性子元素。

❗注意

  • 弹性容器外及弹性子元素内是正常渲染的。
    • 弹性盒子只定义了弹性子元素，
      • 如何在弹性容器内布局。
  • 也就是说，弹性容器的设置只对自己有效，
    • 对其中的子元素和外部元素无效。

弹性盒模型

  • 🔱前言

    • 布局的传统解决方案
      • 基于盒模型，依赖display + position + float，
        • 它对于那些特殊的布局非常不方便。
        • 🍒例如
          • 垂直居中就不容易实现。

启用弹性盒

🔉说明

  • flex：将元素作为弹性伸缩盒显示。
  • inline-flex：将元素作为内联块级弹性伸缩盒显示。

  • ⏳详述

    • 当弹性容器没有设置宽度大小限制时，
      1. 当display指定为flex时，
        • 弹性容器的宽度
          • 会填充父容器 。
      2. 当display指定为inline-flex时，
        • 弹性容器的宽度
          • 会包裹子元素。
  • 🔥案例
  • display: flex;

  • display: inline-flex;

  • ⏳详述

    • 当弹性容器有设置宽度大小限制时，
      1. display指定为flex。
        • 等价于
      2. display指定为inline-flex。

  • 🔥案例

    • display: flex;＆ display: inline-flex;

CSS word-wrap 属性

  • ✨作用
    • 指定长单词或URL地址的换行规则。
  • 🎯特性
默认值 normal
继承 yes
JavaScript 语法 object.style.wordWrap=”break-word”
  • 语法
  • 属性值
值 描述
normal 使用浏览器默认的换行规则。
break-word 允许长单词或URL地址进行换行。

💦另，一行单词中，
实在没有其他靠谱的换行符font>的时候，
才进行换行。

👻特别的

  • 由于word-break和word-wrap长得比较像，
    • 而且属性值也有类似之处。
  • 所以在CSS3规范里，这个属性的名称被修改了，
    • 叫作 overflow-wrap。
  • 虽然这个新属性名称修改了，
    • 显得语义更准确，也更容易区别和记忆。
  • 但是只有Chrome和 Safari等
    • WebKit内核的浏览器支持这个新属性。
  • 因此，虽然换了个好看好用的新名字，
    • 但是为了兼容性，目前还是乖乖地使用word-wrap 吧。
  • 🔥案例
    • word-wrap: break-word;

👁️‍🗨️分析

  • 文本都不会溢出。

🪄总结

  • word-wrap: break-word;的作用是
    • ①如果这一行文字有换行符,
      • 则换行。
    • ②当一个单词，
      • 一整行都显示不下时，
        • 则换行。
  • 解决方案

  • 🔥案例
    • flex-wrap: wrap-reverse;

  • 🔥案例
    • flex-direction: column; +flex-wrap: wrap;

③CSS flex-basis 属性

  • ✨作用

    • 用于指定弹性子元素
      • 在主轴<main axis>方向上的初始宽度。
    • ❗注意
      1. 此初始宽度又称为伸缩基准值。
      2. 如果主轴方向改变
        • 则flex-basis为初始高度。

  • 🎯特性

默认值 auto
继承 否
JavaScript 语法 object.style.flexBasis=”200px”
  • 语法
  • 属性值
值 描述
number 一个单位或者一个百分比。

用于指定弹性子元素，
在主轴<main axis>方向上的初始宽度。
auto 初始宽度等于弹性子元素的宽度。

如果该项目未指定长度，
则初始宽度将由内容决定。
具体分析

  • flex-basis: auto / 0%~100% / 数值;

    • flex-basis: auto;
      • 自动计算。
    • flex-basis: 0%;
      • 设置弹性子元素的宽度为
        • 弹性容器宽度的百分比值。
      • 弹性子元素没有设置width时，
        • 表示弹性子元素的宽度由内容撑开。
    • flex-basis: 120px;
      • 表示设置弹性子元素的初始宽度为120px；

  • 🔥案例

    • 设置第二个弹性子元素的初始宽度为 80px。

  • 🔥案例
    • 设置第二个弹性子元素的初始高度为 80px。

④CSS flex-grow 属性

  • ✨作用

    • 用于指定弹性子元素，
      • 在主轴<main axis>方向上的扩展比例。

  • 🎯特性

默认值 0
继承 否
JavaScript 语法 object.style.flexGrow=”5”
  • 语法
  • 属性值
值 描述
number 默认值是 0。

一个非负数，指定弹性子元素，
在主轴<main axis>方向上的扩展比例。
具体分析

  • ①flex-grow: 0;<默认值>

    • 表示该弹性子元素正常显示宽度，
      • 针对弹性容器多出来的空间不进行处理。

  • 🔥案例

  • ②flex-grow: 1 / 2 / ...;

    • 只给  一个弹性子元素
      • 添加flex-grow属性时，
        • 则将**该行**多余的空间
          • 全部分给该弹性子元素。
    • 🍒例如
      • 只给文本2设置flex-grow：1;

  • 🔥案例

👁️‍🗨️分析

  • 文本2将该行多余的空间占满，
    • 此时所有弹性子元素的总宽度，
      • 和弹性容器相等。

  • ③**同时*给多个弹性子元素*

    • 添加flex-grow属性时，

      • 将**该行**多出来的空间
        • 除以flex-grow属性值的总和，
          • 再按份数<flex-grow属性值>分给盒子弹性子元素。

    • 🍒例如

      • 给文本2添加flex-grow：1;
      • 又给文本3设置了flex-grow：2;
        • 则空间基数的计算方法为：多出来的空间 /（1+2），
        • 文本2占1份，文本3占2份。

⑤CSS flex-shrink 属性

  • ✨作用

    • 用于指定弹性子元素，

      • 在主轴<main axis>方向上的收缩比例。

    • 🪶即当弹性子元素总宽度超过弹性容器的宽度时，

      • 对超出的宽度<溢出总量>进行处理。

  • 🎯特性

默认值 1
继承 否
JavaScript 语法 object.style.flexShrink=”5”
  • 语法
  • 属性值
值 描述
number 默认值是 1。
一个非负数，指定弹性子元素，
在主轴<main axis>方向上的收缩比例。

❗注意

  • 弹性子元素仅在其宽度之和
    • 大于弹性容器的时候才会发生收缩。
      • 其收缩的大小是依据 flex-shrink 的值。
具体分析
  • ① flex-shrink: 1; <默认值>
  • 表示浏览器对弹性子元素超出部分自动处理
    • 此时在弹性盒中，弹性子元素不会自动换行。
  • 当子级元素总宽度超出弹性容器的宽度时，
    • 会自动进行挤压。
  • 🍒例如
    • 超出部分被平均分配给四个弹性子元素，
      • 弹性子元素内部进行了挤压。

  • ② flex-shrink: 0;
    • 浏览器对弹性子元素不进行挤压，
      • 该元素正常显示大小。
    • 将超出空间分配给
      • 其他弹性子元素挤压。
    • 🍒例如
    • 给文本2设置flex-shrink：0；

  • ③给多个弹性子元素添加flex-shrink属性时
  • 🔥案例
    • flex-direction: row;

  • ①如果弹性子元素没有
    • 显示定义fiex-shrink属性，
  • 那么弹性子元素将会自动按照的默认值1，
    • 来计算收缩比例。
  • ②本例中A、B、C显式定义了flex-shrink为1，
    • D、E定义了 flex-shrink为2，
  • 所以计算出来总共将剩余空间分成了7份，
    • 其中A、B、C占1份，D、E占2份，即1 : 1 : 1 : 2 : 2。

  • 我们可以看到弹性容器定义为500px，

    • 弹性子元素被定义为120px，
