模板语法🏅①-1

Vue.js 使用了基于 HTML 的模板语法，允许开发者声明式地将 DOM 绑定至底层 Vue 实例的数据。

知识导读

  • 🍬解析
    1. @click="handleChange()"
      • 为点击事件绑定处理函数。
    2. methods
      • 用于定义事件处理函数。
    3. vm = this
      • 均代表Vue实例对象。

  • 🍬解析
    1. :class="whichcolor"
      • 属性的动态绑定 ➾ whichcolor = data中的whichcolor 。
    2. data: { whichcolor: 'red' }
      • 因为在JS中，纯英文为 ➾ 变量，所以“red”要加引号。

  • 🍬解析
    • :class="isColor?'red':'blue'"
      • 因为在JS中，纯英文为 ➾ 变量，所以“red”和“blue”要加引号。

插值

数据绑定最常见的形式就是使用双大括号语法 ➾ {{...}}的文本插值。

  • 🔔提示
    • 由于双大括号语法提供JS语境，故而可以执行简单的JS代码。

指令🏅②-1

指令是指带有v-前缀的特殊属性，它赋予 HTML 标签额外的功能。

  • ✨作用
    • 当表达式的值改变时，将其产生的连带影响，响应式地作用于 DOM上。

v-show

用于根据表达式的值来{条件性地}显示或隐藏元素。

  • 🔥案例

v-if

用于根据表达式的值来{条件性地}渲染元素或组件。

  • 🔥案例

v-else

给 v-if 添加一个“else”块。

  • 🔥案例

    • 随机生成一个数字，判断是否大于0.5，然后输出对应信息。

v-else-if

给 v-if 添加一个“else-if”块，可以进行多次的{链式}使用。

  • 🔥案例

    • 根据分数来判断等级。

v-show与v-if的异同

共同点

  1. 用法相同

  2. 作用效果相同 ➾ 均能控制元素，在页面上是否显示。

    1. 当表达式为true时，
      • 都会占据页面的位置。
    2. 当表达式为false时，
      • 都不会占据页面位置。
不同点
  1. 控制手段不同
    1. v-show
      • 实际上是为dom元素添加或移除display: none;属性，dom元素还在页面上。
    2. v-if
      • 实际上是将dom元素添加或移除，dom元素不在页面上。
  2. 编译过程不同
    1. v-show
      • 只是简单的基于Css的切换。
    2. v-if
      • 切换时，有一个局部编译/卸载的过程，
        • 它会确保在切换过程中，
        • 条件块内的事件监听器和子组件适当地 ➾ 被销毁和重建。
  3. 编译条件不同
    1. v-show不管条件的真假，始终在dom中，创建处dom元素。
      • 如果条件为假时，不显示元素 ➾ 添加display: none;
      • 如果条件为真，显示元素 ➾ 移除display: none;
    2. v-if是真正的条件渲染。
      1. 如果条件为假时，不做渲染操作 ➾ v-if的懒惰性
      2. 如果条件为真，做渲染操作。
v-show与v-if切换过程中，所触发的组件的生命周期
  1. v-show
    • 由false变为true时，不会触发组件的生命周期。
  2. v-if
    1. 由false变为true时，
      • 触发组件的beforeCreate、create、beforeMount、mounted钩子。
    2. 由true变为false时，
      • 触发组件的beforeDestory、destoryed方法。
v-show与v-if的性能消耗
  1. v-if有更高的切换消耗，
  2. v-show有更高的初始渲染消耗。
v-show与v-if的使用场景🏅②-2
  1. 如果需要非常频繁地切换，则使用 v-show 较好。
  2. 如果在运行时，条件很少改变，则使用 v-if 较好。

v-html🏅③-1

用于输出 html 代码。

  • 🔔提示

    • 双大括号语法会将数据解释为普通文本，而非 HTML 代码。

    • 💫示例

  • 🔥案例

    你的站点上动态渲染的任意 HTML 可能会非常危险，

    • 📌原因
      • 它很容易导致 XSS 攻击。
    • 请只对可信内容使用 HTML 插值，绝不要对用户提供的内容使用插值。

v-for

用于根据数组或对象的属性值来循环渲染元素或组件。

v-model

用于在表单控件和 Vue 实例的数据之间创建双向数据绑定。

  • ✨作用

    • 在 input、select、textarea、checkbox、radio等 表单控件元素上，
      • 创建双向数据绑定，根据表单上的值，自动更新绑定元素的Value值。
